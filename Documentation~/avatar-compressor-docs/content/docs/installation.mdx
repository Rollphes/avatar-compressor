## ALCOM

[ALCOM](https://vrc-get.anatawa12.com/) is the recommended package manager for VRChat avatar development.

<Steps>
<Step>
### Add the Repository

Click the button below to add the Limitex VPM repository:

<VPMRepositoryLink
  repoUrl="https://vpm.limitex.dev/index.json"
  label="Add to ALCOM"
  method="alcom"
/>

Or manually add the repository:

1. Open ALCOM and go to **Settings**
2. Select the **Packages** tab
3. Click **Add Repository**
4. Enter the URL: `https://vpm.limitex.dev/index.json`
5. Click **Add**

</Step>
<Step>

### Install the Package

1. Open ALCOM and select your project
2. Find **Avatar Compressor** in the package list
3. Click **Add** to install

</Step>
</Steps>
</Tab>

## vrc-get

[vrc-get](https://github.com/vrc-get/vrc-get) is a command-line client for VRChat Package Manager.

<Steps>
<Step>
### Add the Repository

```bash
vrc-get repo add https://vpm.limitex.dev/index.json
```

</Step>
<Step>
### Install the Package

Navigate to your Unity project directory, then run:

```bash
vrc-get install dev.limitex.avatar.compressor
```

</Step>
</Steps>

## Manual Installation

<Callout type="warn">
  Manual installation is not recommended. Use a package manager for easier
  updates.
</Callout>

<Steps>
<Step>
### Download the Package

Download the latest `.unitypackage` from the [GitHub Releases](https://github.com/Limitex/avatar-compressor/releases) page.

</Step>
<Step>
### Import to Unity

1. Open your Unity project
2. Drag the `.unitypackage` file into the Unity window, or use **Assets > Import Package > Custom Package**
3. Click **Import** in the dialog

</Step>
</Steps>

## Dependencies

<Callout type="info">
  The following packages will be installed automatically as dependencies.
</Callout>

| Package              | Minimum Version |
| -------------------- | --------------- |
| VRChat SDK - Avatars | 3.10.0          |
| NDMF                 | 1.10.0          |

### NDMF Installation

NDMF (Non-Destructive Modular Framework) is provided via the [Modular Avatar](https://modular-avatar.nadena.dev/) VPM repository. If the installation fails due to NDMF not being found, please add the Modular Avatar repository:

<VPMRepositoryLink
  repoUrl="https://vpm.nadena.dev/vpm.json"
  label="Add Modular Avatar Repository"
  eventName="add_repository_ndmf"
/>

Or manually add the repository URL: `https://vpm.nadena.dev/vpm.json`
